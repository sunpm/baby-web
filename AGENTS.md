# baby-web

## Local UI guardrails
- For any UI, layout, spacing, typography, or interaction change in this repo, read and follow `skills/tool-h5-compact/SKILL.md` first.
- Treat the local skill as the source of truth for this project when it conflicts with external design skills.

## Product constraints
- This app is a tool-first mobile PWA, not a marketing page.
- Prioritize one-hand use, quick logging, low-light readability, and high information density on narrow screens.
- Keep visual hierarchy calm and compact; only the main action area may feel large.
- Avoid oversized hero sections, oversized cards, and decorative whitespace that steals vertical space.
- Prefer existing stack and components; do not add a UI library unless it is already installed or clearly necessary.
- No emojis in UI copy, code, or alt text.
